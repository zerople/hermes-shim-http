# Release notes — v0.1.7

Released: 2026-04-17

## Summary

`v0.1.7` stabilizes the Claude path for real Hermes usage.

The main outcomes are:
- Claude session reuse now works across turns.
- Resume requests no longer resend the large shim system prompt.
- Structured logs are easier to read in normal operation.
- Version metadata, docs, and release artifacts are aligned for publish.

## Highlights

### Claude sessions actually resume now
Fresh Claude requests still start a new session, but follow-up turns can now reuse the best matching prior session with:

```bash
claude -p --resume <old_session_id> --fork-session --session-id <new_session_id>
```

That keeps context continuity while allowing the shim to remain stateless at the HTTP surface.

### Resume turns send only the delta
For fresh sessions, the shim still seeds Claude with the full shim system prompt plus transcript.

For resumed sessions, stdin now contains only the delta transcript suffix instead of the full 20k+ system prompt again. This sharply reduces `stdin_bytes`, repeated prompt work, and timeout risk.

### Cleaner telemetry
Normal logs now focus on the events that matter during operation:
- `chat_completions_request`
- `responses_request`
- `cli_dispatch`
- `session_plan`
- `session_recorded`

Deep per-candidate session-cache mismatch logs are still available, but only when explicitly enabled:

```bash
HERMES_SHIM_HTTP_DEBUG_SESSION_CACHE=1 node ./bin/hermes-shim-http.js ...
```

## Upgrade notes

- Update to `0.1.7` before testing Claude resume behavior.
- If you are verifying session reuse, look for:
  - `resume_used: true`
  - `append_system_prompt_used: false`
  - sharply smaller `stdin_bytes` on resumed turns

## Maintainer checklist

Before publishing:

```bash
python3 -m pytest -q
node --test tests/*.test.js
npm pack
npm publish --dry-run --access public
```

For this release, manual package publication is acceptable after the git push is complete.
